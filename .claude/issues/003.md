## 概要
現状の漏れトークン検知(シグネチャ1)は `LEAK_WORDS_RE` をフック本体に直書きしており、新しい
迷子トークンが観測されるたびにフック本体の編集が必要になる(issue 002 の county がその実例)。
これを「**外部設定ファイルに列挙したトリガー文字列のどれかに単独行でマッチしたら継続を促す**」
という汎用マッチャに作り変え、トークン追加を「設定ファイルに1行足すだけ」にする。
あわせて、本リポジトリ版(OSS クローン)が個人運用版より遅れている乖離も解消する。


## 出典
- ユーザ提案(2026-06-21): 「特定の文字列にマッチしたら『続けてください』を出す汎用構造にし、
  その文字列群を別の設定ファイルに配列等で記述。デフォルトには既知のトークンを同梱したい」。
- 親 issue 001(初回実装)、002(county 追補)。


## 詳細
### 決定済みの設計方針(ユーザ確認済み)
1. **設定ファイル形式**: 1トークン1行のプレーンテキスト。`#` 始まりはコメント。`jq` 不要で
   `grep -f` で直接読める。トークンは**リテラル文字列**として扱う(正規表現メタ文字を解釈しない)。
   配布する既定ファイルの例:
   ```
   # toolparse_recovery: leak tokens (one per line)
   # lone-line, case-insensitive, whole-line match
   court
   county
   course
   cource
   call
   # add new tokens below as you observe them
   ```
2. **デフォルトの扱い**: 配布ファイルに既知デフォルトを最初から同梱する(**置換セマンティクス**=
   ファイルが単一の真実)。設定ファイルが存在しない/読めない場合は、スクリプト組込みの
   デフォルト(現行と同じ court/county/course/cource/call)へ**フォールバック**して必ず動作する。
3. **外部化の対象はシグネチャ1のみ**。シグネチャ2(構造検知: stop_reason=tool_use かつ
   tool_use ブロック無し)とシグネチャ3(諦め定型文 + stop_sequence)は「トークン一致」ではなく
   transcript の stop_reason を見る別種の検知であり、汎用文字列リストに混ぜると誤検知ガードが
   壊れる(特に T4: 諦め文を end_turn で全文出力した本物の応答を block しない、が破綻する)。
   よって両者は専用検知のまま据え置く。

### 維持すべき誤検知ガード(重要)
- マッチ条件は現行同様「**単独行・全体一致・前後空白のみ許容・大文字小文字無視**」を厳守する。
  文中出現(例: `Fairfax county のデータを集計しました。`)では発火させない。
  既存テスト T6/T7 相当のケースで担保する。
- 部分一致は採用しない(誤検知が激増するため)。

### 本リポジトリ版と個人運用版の乖離(同時解消)
調査の結果、個人運用版フック `~/.claude/hooks/toolparse_recovery.sh` は本リポジトリ版に無い
以下を持ち、OSS クローンへの反映が漏れている:

| 機能 | 個人版 | 本リポジトリ版 |
|---|---|---|
| シグネチャ1(漏れトークン) | あり | あり |
| シグネチャ2(構造検知) | あり | あり |
| シグネチャ3(諦め定型文+stop_sequence) | あり | なし |
| recovered イベント記録(block後の解消=成功を定量化) | あり | なし |
| CLAUDE_TOOLPARSE_LOG(ログ出力先上書き) | あり | なし |
| reason 文言 | 日本語 | 英語(Please continue.) |

本 issue で両版を作り直す好機なので、個人版の機能(シグネチャ3 / recovered / CLAUDE_TOOLPARSE_LOG)を
本リポジトリ版へ反映し、ロジックを一致させる。reason 文言は公開版=英語/個人版=日本語の方針を維持するか、
変数化して切替可能にするかは実装時に検討。

```mermaid

    A[Stop hook 発火] --> B{単独行が tokens ファイルの<br/>いずれかに全体一致?}
    B -- はい --> D[検知: 継続を促す]
    B -- いいえ --> C{構造検知 or 諦め定型文?}

    C -- いいえ --> Z[クリーン停止]
    subgraph 外部化対象
      B
    end
    subgraph 据え置き(専用検知)
      C
    end
```


## 方針
- フック本体は「tokens ファイル読込 → 単独行・全体一致でトリガー判定」に汎用化。
- tokens ファイルのパスは `${CLAUDE_CONFIG_DIR:-$HOME/.claude}/toolparse_recovery.tokens`(可搬性のため)を想定。
  本リポジトリに既定 tokens ファイルを同梱し、`install.sh` で未存在時のみコピー(既存をtokenファイルを上書きしない冪等性)。
- 実装候補: `grep -iqE` で動的にalternationを組む / `grep -iFxq -f <tokens>`(リテラル全体一致)で読む 等。
  空白トリム・コメント行(#)・空行の除外を考慮する。実装時に最小で確実な方式を選定。
- TDD: 既存テスト(単独行検知・文中非検知・構造検知・諦め文・recovered)を維持しつつ、
  「設定ファイルにトークンを足すと検知される」「設定ファイル不在時は組込デフォルトで動く」テストを追加。
